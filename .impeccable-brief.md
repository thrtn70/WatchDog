# WatchDog Full App Redesign — Design Brief

## 1. Feature Summary
Complete visual and layout redesign of WatchDog's main window — clip library, preview/editor panel, toolbar, status bar, and floating overlays. Replace the generic Catppuccin Mocha theme with a purpose-built WatchDog identity: custom dark palette anchored by cyan/teal accent, denser information display, switchable view modes for 20-50+ clip libraries, and controls that feel crafted. The dog skull brand mark appears subtly as part of the app's character.

## 2. Primary User Action
**Browse recent clips after a gaming session.** The user has just finished playing, opens WatchDog, and wants to quickly scan 20-50+ thumbnails, identify good moments, and share or lightly trim. The clip library IS the app.

## 3. Design Direction

**Palette:** Custom dark theme with cool-toned neutrals tinted toward cyan/teal. Signature accent: sharp, saturated cyan/teal — used sparingly for active states, selection, recording indicators, primary actions. Semantic colors: teal for idle/success, warm amber for recording, red for destructive, muted variants for secondary.

**Typography:** Bundle a custom font with more personality at small sizes than Segoe UI. Target: "precision tool" feel at 11-13px. Consider Recursive (variable, mono-capable), Geist, or Manrope.

**Tone:** Confident and slightly opinionated. Tighter spacing, sharper corners where appropriate, bolder contrast between active/inactive states. Feels like it was designed by someone who uses it daily.

**Brand presence:** Dog skull in empty state illustration, subtle watermark/motif, tiny mark in titlebar. Easter egg you notice, not mascot being pushed.

## 4. Layout Strategy

**3 zones, top to bottom:**
1. **Command bar** (~36-40px): Title/logo, view toggle, game filter, sort, search, gear icon. Dense, single-row.
2. **Content area** (fills space): Clip library left (dominant, ~65%), editor panel right (~35%, collapses when no clip selected).
3. **Status strip** (24px): Recording state, capture status, storage usage. One line.

**Clip library views (switchable):**
- **Grid view** — Thumbnail cards, 3 density options (compact/default/large)
- **List view** — Rows with small thumbnail, scannable for 50+ clips, sortable columns
- **Filmstrip view** (stretch) — Horizontal scrolling strip

**Editor panel:** Appears on clip selection, collapses when deselected. Video preview, timeline with trim handles, metadata, action buttons.

**Floating panels:** Hidden by default. Accessible via command bar or keyboard shortcut. Overlay content area when opened.

## 5. Key States

| State | What user sees | Tone |
|-------|---------------|------|
| Empty library | Dog skull illustration, "No clips yet — start a game and press F9" | Welcoming |
| Browsing clips | Grid/list with thumbnails, game badges, timestamps | Efficient, scannable |
| Clip selected | Editor panel slides in, metadata visible, actions ready | Focused |
| Recording active | Amber indicator in status strip and tray, subtle pulse/dot | Confident |
| Sharing to Discord | Compact toast or inline progress, not a modal | Fast, non-blocking |
| Trim in progress | Timeline highlights trim region, progress on export button | Smooth |
| Error | Red-tinted inline message near failed action, no modal | Clear, not alarming |
| First launch | Setup guidance: "Launch a game and WatchDog starts automatically" | Helpful |

## 6. Interaction Model

- **View switching:** Toggle icons in command bar, instant switch
- **Clip selection:** Single click = select + open editor. Double-click = play.
- **Context menu:** Right-click for quick actions
- **Keyboard:** Arrow keys navigate, Enter plays, Delete removes, Ctrl+D shares
- **Drag and drop:** Drag clip out of window to export file
- **Search/filter:** Type-to-filter, game dropdown, sort dropdown, all instant
- **Floating panels:** Toggle via icon/shortcut, draggable, snap to edges, persist position

## 7. Content Requirements

**Labels:** "WatchDog" with dog skull icon, view toggle icons, "Filter clips..." placeholder, relative timestamps ("2m ago"), "Idle"/"Recording CS2"/"Buffer active", "42 clips · 3.2 GB"

**Dynamic ranges:** 0-500+ clips, 100+ games, 5s-30min durations, 10MB-2GB sizes, seconds-to-months timestamps

**Error messages:** Specific and actionable, never generic.

- View mode should persist across sessions
- Settings window out of scope for this brief (shape separately)
